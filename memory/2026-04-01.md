### Topological Sort II (LC210) 核心模式
- 与LC207相同,但需要收集弹出顺序到order数组
- BFS: indegree==0入队→弹出→收集到order→邻居indegree--
- 最后检查orderSize==numCourses

### LIS Binary Search 核心模式
- tails[i] = 长度为i+1的LIS的最小尾元素(递增序列)
- 二分查找: 找第一个>=nums[i]的位置
- 如果插入位置==size,说明找到更长的LIS,size++
- 时间O(n log n), 空间O(n)

### LCS DP 核心模式
- dp[i][j] = text1前i个与text2前j个的LCS长度
- 字符相等: dp[i][j] = dp[i-1][j-1] + 1
- 字符不等: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
- 滚动数组优化: 只用两行,交替使用
- 时间O(mn), 空间O(min(m,n))

## 晚间补充 (19:44) - Segment Tree & BIT 专项突破

### 新增LeetCode专题: Segment Tree + Fenwick Tree

| 题目 | 难度 | 算法 | 核心洞察 |
|------|------|------|----------|
| LC307 Range Sum Query - Mutable | Medium | Segment Tree (Iterative) | 迭代式: 叶子节点在下标size+idx, 父节点不断向上更新 |
| LC315 Count of Smaller Numbers | Hard | BIT + Coordinate Compression | 从右往左遍历, BIT查询当前数前有多少更小数 |
| LC493 Reverse Pairs | Hard | BIT + Coordinate Compression | 从左往右遍历, query(2*nums[i])右半部分count |

### BIT核心操作
```
Update(idx, delta):    while(idx <= n) { bit[idx] += delta; idx += idx & -idx; }
QueryPrefix(idx):      while(idx > 0)  { sum += bit[idx];    idx -= idx & -idx; }
RangeSum(l, r):        QueryPrefix(r) - QueryPrefix(l-1)
```

### Segment Tree vs BIT 对比
| 特性 | Segment Tree | BIT |
|------|-------------|-----|
| 时间(单点更新+区间查询) | O(log n) | O(log n) |
| 时间(区间更新+单点查询) | O(log n) | O(log n) |
| 空间 | 4n | n+1 |
| 代码复杂度 | 较高 | 极简 |
| 灵活性 | 支持任意区间合并 | 仅支持前缀可合并操作 |

### Coordinate Compression 模式
```c
// 对nums做坐标压缩:
// 1. 复制nums到sorted, 排序去重得到uniq[]
// 2. 二分查找每个nums[i]在uniq中的下标作为rank
// 3. BIT/rank tree 用 rank 作为索引 (1-indexed)
// 关键: 确保2*nums[i]也在压缩空间中 (LC493)
```

### 315 Count of Smaller Numbers 核心模式
```
1. nums从右往左遍历
2. BIT记录已遍历过的数(频率)
3. res[i] = BIT.query(rank(nums[i]) - 1)  // 比nums[i]小的数有多少
4. BIT.add(rank(nums[i]), 1)               // 把当前数加入BIT
```

### 493 Reverse Pairs 核心模式
```
1. nums从左往右遍历
2. 压缩空间要包含2*nums[i]的值
3. res += BIT.sum(m-1) - BIT.sum(rank(2*nums[i]))  // 比2*nums[i]大的数
4. BIT.add(rank(nums[i]), 1)
```

### State DP 核心模式

**House Robber (打家劫舍)**:
```
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
空间优化: prev2=nums[0], prev1=max(nums[0],nums[1])
```

**House Robber II (环形)**:
```
两种情况:
1. 偷第一间, 不偷最后一间 → houses[0..n-2]
2. 不偷第一间, 偷最后一间 → houses[1..n-1]
取max(情况1, 情况2)
```

**Stock Sell Buy 系列 (多状态)**:
```
k=2时4状态:
buy1  = max(buy1,  -price[i])
sell1 = max(sell1, buy1  + price[i])
buy2  = max(buy2,  sell1 - price[i])
sell2 = max(sell2, buy2  + price[i])

k=任意时:
buy[j]  = max(buy[j],  sell[j-1] - price[i])
sell[j] = max(sell[j], buy[j]   + price[i])
```

### Interval DP 核心模式

**Decode Ways**:
```
dp[0]=1 (空字符串=1)
dp[1]=1 if s[0]!='0' else 0
dp[i] += dp[i-1] if s[i-1]!='0'
dp[i] += dp[i-2] if s[i-2..i-1] in [10,26]
```

**Unique BSTs (Catalan)**:
```
C_0=1, C_1=1, C_2=2, C_3=5, C_4=14, C_5=42
C_n = sum_{i=0}^{n-1} C_i * C_{n-1-i}
```

**Palindrome Partitioning**:
```
1. 预计算isPal[i][j]: s[i]==s[j] && (j-i<2 || isPal[i+1][j-1])
2. dp[i] = 0 if isPal[0][i]
3. dp[i] = min(dp[j] + 1) for j in [0,i) where isPal[j+1][i]
```

**Dungeon Game (逆序DP)**:
```
从右下角向左上角推导:
need = min(next_row, next_col)
dp[i][j] = max(1, need - dungeon[i][j])
最终答案: dp[0][0]
```
