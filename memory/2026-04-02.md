## 经验沉淀

### Bash 脚本关键技巧
1. `set -e` 下 `((var++))` 当 var=0 返回 1 → 改用 `((var++)) || true`
2. bash 3.2 不支持 `${var^}` → `sed 's/./\U&/' | awk '{print toupper(substr($0,1,1)) tolower(substr($0,2))}'`
3. macOS awk (mawk) 不支持 `match(s, r, arr)` 三参数 → `grep -Eo 'pattern\(' | sed 's/(//'`
4. `[[ expr ]] && exit 1` 中 expr 为假时 `[[ ]]` 返回 1 → 分开检查

### Playwright WASM 测试
- `file://` 协议下 WASM 的 XHR fetch 被 CORS 阻止
- 解决: Playwright beforeAll 启动 Node.js HTTP server
- `NODE_PATH` 指向 builder 的 node_modules 以使用本地 @playwright/test

### Emscripten WASM 加载
- `mahjong.js` 已定义 `var Module` 对象
- 不能重新定义 `Module`，会覆盖已定义的 `onRuntimeInitialized` 等
- 正确方式：在引入 `mahjong.js` **之前** 定义 `var Module = { onRuntimeInitialized: fn }`
- Emscripten 会在运行时初始化完成后自动调用 `onRuntimeInitialized`
