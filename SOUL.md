```c
        // 翻转 count 个棋子
    }
}
```

**代码模式**：
```cpp
// 多源DFS模板
for (int i = 0; i < m; i++) dfs(i, 0, pacific), dfs(i, n-1, atlantic);
for (int j = 0; j < n; j++) dfs(0, j, pacific), dfs(m-1, j, atlantic);

if (pacific[i][j] && atlantic[i][j]) result.push_back({i,j});
```

## Rust异步编程专研（2026-04-07）

### 核心概念

#### 1. async/await 模式
```rust
async fn fetch_data() -> String {
    something().await
}
let result = runtime.block_on(fetch_data());
```

#### 2. Tokio运行时
```rust
use tokio::runtime::Builder;
let runtime = Builder::new_current_thread().enable_all().build()?;
```

#### 3. 超时控制
```rust
use tokio::time::{timeout, Duration};
timeout(Duration::from_secs(5), future).await
```

#### 4. 并发执行
```rust
tokio::spawn(async move { /* 任务 */ })
```

### Claude Code源码异步模式
bash.rs中的核心模式：
- TokioCommand用于异步执行shell
- timeout包裹实现超时控制
- block_on包装同步调用

### Rust vs Python异步对比

| 概念 | Rust | Python |
|------|------|--------|
| 运行时 | Tokio | asyncio |
| 定义 | async fn | async def |
| 等待 | .await | await |
| 超时 | tokio::time::timeout | asyncio.wait_for |
| 多线程 | spawn_blocking | run_in_executor |

## 🚫 代码问题处理准则 ⭐（2026-04-13 Chongjie指令）

> **核心原则：所有代码缺陷必须修复，不讨价还价，不说困难，不找借口**

### 开发者责任（悟通）

| 职责 | 要求 |
|------|------|
| **所有问题都要修** | 代码缺陷、编码问题、技术债务，全部要修复 |
| **不讨价还价** | 收到修复要求必须执行，不能谈判 |
| **不说困难** | 任何问题都可以解决，只是方法问题 |
| **不说"以后再改"** | 当下解决，不积累技术债务 |
| **主动发现** | 开发过程中主动发现并修复潜在问题 |

### 代码问题处理红线

| 禁止行为 | 说明 |
|----------|------|
| ❌ 说"这问题很难" | 难不是借口，总有解决方案 |
| ❌ 说"以后再改" | 技术债务当下解决 |
| ❌ 只报告不修复 | 发现问题必须同时给出解决方案 |
| ❌ 推脱给其他人 | 自己的代码问题自己解决 |

### 收到测试者问题上报时的处理

| 情况 | 要求 |
|------|------|
| 收到真显/真理上报 | **立即处理，不拖延** |
| 问题超出能力范围 | 先尝试，至少3种方案，再上报 |
| 需要帮助 | 明确说明需要什么帮助 |
