## 六、任务分类与处理规则

### 6.1 任务分类

| 任务类型 | 定义 | 处理方式 |
|----------|------|----------|
| **普通任务** | 日常Routine、独立可完成 | agent执行→汇报SC→完成 |
| **授权任务** | 主人明确说"授权XXX" | 被授权agent完成本阶段→**主动汇报SC并触发下一步** |
| **协作任务** | 同岗Agent协同完成 | Agent间直接协作→汇报SC |

### 6.2 普通任务处理流程

```
主人指令 → SC分解/Jaden协助分解 → 分配Agent → Agent执行 → 汇报结果 → SC聚合 → 完成
                              ↑
                    （超时应答或失败）→ 重新分配/升级
```

### 6.3 禁止懒委托规则 ⭐

> SC给Agent的每条指令必须包含以下4要素，**禁止只给方向性描述**

| 要素 | 必须包含 | 示例 |
|------|----------|------|
| **文件路径** | 精确到文件，可含行号 | `src/gateway/session.py:234` |
| **任务目标** | 具体做什么，不模糊 | "修复RLock未释放" |
| **验收标准** | 可验证的完成条件 | "round31测试通过" |
| **上下文摘要** | ≤200字背景说明 | "原因为RLock在异常路径未释放" |

## 七、汇报机制

### 7.1 统一汇报格式（3行简化版）

```
# [Agent]汇报 - HH:MM
状态: ✅完成 / 🔄进行中 / ❌受阻
产出: [文件路径/关键数据]
下一步: [需要什么？/ 无需下一步]
```

### 7.2 汇报触发规则

| 任务类型 | 汇报时机 | 汇报对象 |
|----------|----------|----------|
| 普通任务 | 完成→立即汇报 | SC |

| 同岗协作 | 协作完成→汇报 | SC（如需SC知晓） |

### 7.3 主动汇报触发条件

| 条件 | 必须汇报 |
|------|----------|
| 任务完成 | ✅ 立即汇报 |
| 任务失败 | ✅ 立即汇报（附原因和重试方案） |
| 任务超时 | ✅ 预警（距超时5分钟前） |
| 发现重大风险 | ✅ 立即上报 |

## 十四、任务历史管理协议 ⭐

### 核心原则

**每个Agent负责管理自己任务的历史记录**，遵循以下规则：

| 状态 | 定义 | 处理方式 |
|------|------|----------|
| **活跃** | 任务进行中、需要跟踪 | 保留在原文件 |
| **已解决** | 问题已修复、任务完成 | 移动到归档目录 |
| **已过期** | 条件不再适用、时间已过 | 移动到归档目录 |
| **已失效** | 前置条件不再有效 | 移动到归档目录 |

### 归档目录结构

```
{AGENT_WORKSPACE}/memory/archive/
├── YYYY-MM/                    # 按年月分类
│   ├── task-archive-YYYY-MM-DD.md
│   └── issue-archive-YYYY-MM-DD.md
└── index.md                    # 归档索引
```
